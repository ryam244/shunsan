## Phase 4: アップデート

### 4.1 アップデートの種類

#### OTAアップデート（Over-The-Air）
| 項目 | 内容 |
|------|------|
| 適用範囲 | JavaScript/TypeScriptの変更のみ |
| 方法 | `eas update` コマンド |
| 審査 | 不要 |
| 反映時間 | 即時〜数分 |

#### ストアアップデート
| 項目 | 内容 |
|------|------|
| 適用範囲 | ネイティブコード変更、新SDK、新権限 |
| 方法 | 新ビルド → App Store審査 |
| 審査 | 必要（1-3日） |

### 4.2 バージョニング戦略
```
バージョン形式: MAJOR.MINOR.PATCH
例: 1.0.0 → 1.0.1 → 1.1.0 → 2.0.0

MAJOR: 大幅な機能追加・UI変更
MINOR: 新機能追加
PATCH: バグ修正、軽微な改善
```

### 4.3 アップデートサイクル

| フェーズ | 期間 | 内容 |
|----------|------|------|
| リリース直後 | 1-2週間 | バグ修正集中（v1.0.1, v1.0.2） |
| 安定期 | 月1回 | 機能追加・改善（v1.1.0, v1.2.0） |
| 成熟期 | 四半期 | 大型アップデート（v2.0.0） |

### 4.4 アップデート時のチェックリスト
| タスク | 状態 |
|--------|------|
| CHANGELOGの更新 | [ ] |
| バージョン番号の更新（app.json） | [ ] |

| 「新機能」説明文の更新 | [ ] |
